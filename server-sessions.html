Creating Server-Side Sessions

Server-side sessions are a fundamental feature in web development that allow you to maintain state and store user-specific data across multiple web pages. Let's explore how to create single or multiple server-side sessions:

Starting a Session in PHP:

In PHP, you can start a session using the session_start() function. This function must be called before any output is sent to the browser.


// Start a PHP session
session_start();
Creating a Session Variable:

You can store data in the session by using session variables. These variables are accessible across different pages during the same session.


// Store data in a session variable
$_SESSION['username'] = 'John';
Accessing Session Data:

You can access session data by referring to the session variables. This allows you to retrieve user-specific information.


// Access session data
$username = $_SESSION['username'];
Creating Multiple Sessions:

It's possible to create and manage multiple sessions in PHP. Each session is identified by a unique session ID and can store different sets of data.


// Start a second session
session_id('second_session');
session_start();

// Store data in the second session
$_SESSION['user_type'] = 'Admin';
Session Termination:

Sessions should be properly terminated when they are no longer needed. You can use the session_destroy() function to destroy the current session.


// Terminate the current session
session_destroy();
